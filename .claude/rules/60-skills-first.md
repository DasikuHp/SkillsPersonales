# 60 · Skills-first

Antes de escribir código o actuar en un dominio que tiene skill, **lee su `SKILL.md`**. No decidas de
antemano si la tarea "necesita" la skill: las propias skills definen qué cubren. Pueden aplicar varias a
la vez.

Mapa rápido tarea → skill:
- Vídeo / explainer / captions → `video-production`.
- RPG 3D / Godot / GDScript → `godot-rpg`.
- Indexar o consultar el codebase, "¿dónde está X?" → `codebase-memory`.
- Atacar/defender/auditar TU app → `appsec-inward` (respeta `00-security-inward`).
- Decisión difícil, trade-off → `ego-debate`.
- Recordar contexto entre sesiones → `ego-memory`.

Antes de arrancar un motor externo, lee su `ensure` y su `_SOURCE.md` (procedencia y qué se portó).
